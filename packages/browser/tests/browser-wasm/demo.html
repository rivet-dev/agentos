Agent OS in the browser

The Agent OS sidecar, compiled to WebAssembly, booting and answering a real ACP wire request inside this page — the same Rust kernel that runs natively, now in the browser.

    What this proves: the converged agentos-sidecar-browser wasm module decodes a BARE wire frame, runs the authenticate handshake, routes an ACP get_session_state request through BrowserAcpExtension → AcpCore (the host-free engine shared with the native sidecar), and returns a real ACP response — all in-page, with the wasm kernel as the sole enforcement point.

    Running a full agent (e.g. pi) end-to-end in the browser additionally requires the browser agent-process executor (worker stdio wiring) and host network egress; both are tracked as out-of-scope for the current browser-convergence milestone. See DEMO.md.

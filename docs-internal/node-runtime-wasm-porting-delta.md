# EdgeJS to Node 24.15.0 porting delta

Status: **R0 input inventory; no EdgeJS architecture is accepted**

The machine-readable file inventory is
[`node-runtime-wasm-porting-delta.json`](./node-runtime-wasm-porting-delta.json),
generated by `scripts/generate-node-runtime-wasm-porting-delta.mjs`. It compares
the pinned EdgeJS source at
`b1feaa2c2b36f443ee5d527161dd93f3ac1544d6` with Node 24.15.0 at
`848430679556aed0bd073f2bc263331ad84fa119` by exact path, byte length, and
SHA-256.

## Source delta

EdgeJS identifies its Node base as 24.13.2. The target is Node 24.15.0.

| Family | Exact | Modified | EdgeJS-only | Node-only |
|---|---:|---:|---:|---:|
| JavaScript `lib/` | 237 | 114 | 2 | 1 |
| native `src/` by matching path | 0 | 4 | 163 | 431 |

The native count is intentionally not a similarity score. EdgeJS moved most
Node behavior into `edge_*` files and Node-API modules, so matching only four
filenames is evidence that this is a source port, not a patch that can be
applied wholesale. Every target JavaScript modification remains a compatibility
obligation in `node-runtime-wasm-compatibility.json`.

## Architectural delta

The following EdgeJS properties are rejected and must not survive the AgentOS
port:

- Its pluggable-engine and Wasmer/WASIX host architecture. AgentOS uses the
  existing native V8 isolate and instantiates `node-runtime.wasm` inside that
  same isolate.
- Its WASIX compiler, sysroot, compatibility header, and direct WASIX build
  scripts. AgentOS rebuilds against its owned `wasm32-wasip1` sysroot and the
  single generated Linux/POSIX import provider.
- Its guest-specific libc stubs for timezone, `fork`, interface lookup,
  scheduler/thread names, PTYs, process memory reporting, dynamic loading, and
  similar gaps listed in `vendor/edgejs/wasix/WASIX_TODO.md`. Each gap is fixed
  in the AgentOS sysroot, kernel, or generic syscall provider; a stub is not
  parity evidence.
- Its bundled `napi/v8` engine provider and engine-owned system binding model.
  The pinned N-API source is a semantic reference only. AgentOS freezes
  `agentos_napi_v1` as a wasm32 wire ABI and keeps all V8 calls isolate-local.
- Its host allocator and external-backing shortcuts. Copy mode is required;
  pinned external backing remains disabled unless the R0 memory-growth, GC,
  finalizer, and teardown proofs pass.
- Its OpenSSL-WASIX build. AgentOS builds Node's pinned OpenSSL 3.5.5 source
  against the owned sysroot and proves threaded execution through the generic
  pthread/futex surface.

## Porting disposition

Code under `vendor/edgejs/src` may be adapted when it preserves Node behavior
and uses only `agentos_napi_v1`, `agentos_node_engine_v1`, or the generated
Linux/POSIX ABI. Code under `vendor/edgejs/wasix` is an implementation-gap
inventory, not production input. Every imported symbol, stub replacement, and
behavioral deviation must be represented in the generated ABI inventories and
linked to a required compatibility test before R0 can pass.

Node-API addon support remains conditional on the R0 loader proof. Failure of
that proof results in the explicit first-cutover unsupported decision required
by the replacement spec; it never permits host-native addon loading.
